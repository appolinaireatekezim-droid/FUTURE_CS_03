# FUTURE_CS_03

## Task 3 - API Security Risk Analysis

This repository contains my submission for Future Interns Cyber Security Task 3.

## Description

This project focuses on analyzing the security posture of an API by identifying possible risks related to authentication, authorization, data exposure, input validation, and rate limiting. The objective is to review API behavior, detect visible security weaknesses, and provide practical remediation recommendations in a professional report.

The analysis was conducted using safe and non-destructive methods on test or demo APIs. The final report explains technical findings in simple and business-friendly language to demonstrate both security understanding and reporting skills.

## Objectives

- Identify insecure API endpoints and data exposure risks
- Review authentication and authorization weaknesses
- Detect missing validation or rate-limiting controls
- Provide practical remediation suggestions

## Tools Used

- Postman or Insomnia
- Browser Developer Tools
- Public test/demo APIs
- Google Docs / MS Word / PDF

## Repository Contents

- `FUTURE_CS_03.pdf` – Final API security risk analysis report
- `README.md` – Project documentation
- `screenshots/` – Supporting screenshots and evidence (if included)

## Methodology

1. Selected a test or demo API for analysis
2. Reviewed available endpoints and request/response behavior
3. Checked authentication and authorization mechanisms
4. Observed possible data exposure and input validation issues
5. Assessed whether security controls such as rate limiting were visible
6. Documented findings and provided remediation recommendations

## Expected Outcome

This project demonstrates the ability to analyze API-related security risks, document findings professionally, and communicate technical issues in a clear and structured way.

## Disclaimer

This work was conducted strictly for educational purposes as part of the Future Interns Cyber Security Internship. Only safe, non-destructive, and authorized testing methods were used.
